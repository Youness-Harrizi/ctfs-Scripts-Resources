Bypass DNS filters (theory)


Intro

  • rely on block list usualy (e.g malwaredomainlist)
  • when performing DNS lookups our request will pass by DNS server proxy
  • if domain matchs blocked => request could be blocked or returned a feke IP t-hat will show the user a malware error page

Bypass DNS filters

  • chose domain that seems legitimate enough (don't contain random numbers and charcters) because of Natural process language
  • if new domains it may be classified as Newly seen domain => lower score
  • need to match the domain category allowed by the client
    • we can request recategorization e.g via OPENDNS
